# 测试说明

完整检查：

```bash
pnpm lint
pnpm test
pnpm build
cargo test --manifest-path src-tauri/Cargo.toml
```

自动测试覆盖：

- YAML 解析与旧 Markdown 不改写
- 标签、字数、大纲和双向链接转换
- 安全路径与非法文件名
- Markdown 索引、读取和中文标签
- 改名后的双向链接和相对链接更新
- 10,000 篇笔记的 SQLite FTS5 搜索响应
- Word 导入保持字节级一致，并处理 `documents/` 内的同名文件
- 外部源文件到资料库副本的单向同步，不发生反向覆盖
- `.docx` 预览读取失败、损坏文件，以及切换/关闭时的 Blob URL 释放
- `.doc`/`.docx` 交给系统默认应用打开，以及默认应用缺失时的错误提示

发布前手动验收：

1. 同时添加两个资料库并完成索引。
2. 创建普通笔记和每日笔记。
3. 在三种编辑模式间切换并检查公式、Mermaid、表格和任务列表。
4. 粘贴图片，确认进入资料库根目录的 `assets/`。
5. 添加标签、收藏、置顶和双向链接。
6. 重命名被引用笔记并确认链接自动更新。
7. 新建笔记后立即输入内容并重命名，确认内容保留、旧文件不再出现，且输入 `.md`/`.MD` 不会重复扩展名。
8. 使用全局搜索打开另一资料库中的笔记。
9. 用其他编辑器修改已打开的文件，确认出现冲突保护。
10. 等待历史快照生成，重启应用后恢复旧版本。
11. 切换三套主题及明亮、深色、跟随系统模式。

## Word 文档验收

样本文档使用桌面上的：

```text
/Users/henry/Desktop/ChatPDF_FAISS_RAG学习笔记.docx
```

该路径仅用于本机验收，不应写入自动测试、应用默认设置或提交到仓库。验收前先复制一份可修改的源文件到任务专用临时目录，避免改动用户桌面原件；验收结束后删除这份临时副本和所有渲染中间物。

1. 记录样例源文件的大小与校验和，导入当前资料库。
2. 确认生成 `documents/ChatPDF_FAISS_RAG学习笔记.docx`，其字节数和校验和与源文件完全一致，桌面原件未被改写。
3. 在应用内打开 `.docx` 预览，检查正文、标题层级、图片和表格等可识别内容；复杂分页、字体、文本框或公式存在差异时，应视为预览保真限制，而不是修改原文件来迎合预览。
4. 点击“使用本地默认软件打开”，确认源文件可用时系统配置的 Word、WPS 或 LibreOffice 收到外部源文件；移除关联或让源文件不可用后，应改为打开 `documents/` 副本。若没有默认应用，应显示可理解的错误，不崩溃。
5. 搜索完整或部分文件名、`documents/` 相对路径，应能找到文档；搜索只存在于 Word 正文中的词语，不应命中。
6. 修改任务临时目录中的外部源文件并等待同步，确认资料库副本更新且再次保持字节级一致。
7. 直接改动资料库副本，确认应用不会把该变化反向写回外部源文件；下一次源文件变化可覆盖副本，并应有明确状态或提示。
8. 移动或删除临时源文件，确认资料库副本仍可保留和打开，同时界面提示源不可用，且不会误删副本；使用“重新关联”选择新的源文件后可恢复单向同步。
9. 另选一个 `.doc` 文件，确认不进入内嵌预览，而是提供系统默认应用打开。
10. 关闭预览、切换文件并重启应用，确认没有 DOCX 解包目录、HTML、PDF 或 PNG 等运行时磁盘预览产物；通过开发工具或自动测试确认切换、关闭及预览失败路径都会释放 Blob URL。

完成后删除验收资料库中的测试副本以及任务临时目录。只核对墨岛笔记自己创建的临时命名空间，不进行无边界的全磁盘扫描。
