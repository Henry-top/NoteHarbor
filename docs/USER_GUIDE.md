# 墨岛笔记使用说明

## 安装与首次启动

在 macOS 上打开 DMG，把“墨岛笔记”拖到“应用程序”。当前首版没有 Apple 公证；如果系统拦截首次启动，请在访达中右键应用并选择“打开”。

首次进入后点击“添加本地资料库”，选择一个存放 Markdown 的文件夹。可以继续添加更多资料库；新建笔记、每日笔记和附件会保存到当前选中的资料库。

首次启动会显示五步新手引导。鼠标停留在主要图标按钮上会看到功能说明和快捷键；点击窗口右上角的问号或按 `F1`，可以打开可搜索的“使用帮助”，也可以从那里重新查看新手引导。

## 日常使用

- 点击侧栏的“新建笔记”创建带稳定 ID 和时间信息的 `.md` 文件。
- 打开笔记后，使用右上角“…”菜单中的“重命名”打开应用内改名窗口；同库内引用它的 `[[双向链接]]` 和相对链接会随之更新。
- 在侧栏文件上点击右键，可以重命名、复制笔记或链接、置顶、收藏、在系统文件管理器中显示，以及移到废纸篓/回收站；Word 文档还可以直接打开、同步或重新关联源文件。
- `⌘K`（Windows 为 `Ctrl+K`）打开全局搜索，可查找所有已登记资料库。
- 编辑器顶部可切换源码、即时渲染和左右分栏三种模式。
- 右侧面板包含大纲、反向链接和历史版本；面板可以折叠。
- 粘贴图片或拖入附件时，文件会进入当前资料库根目录的 `assets/`。
- 内容停止输入约 500 毫秒后自动保存；稳定约 30 秒后生成历史快照。

## Markdown 扩展

应用支持表格、任务列表、脚注、代码高亮、数学公式、Mermaid、普通相对链接，以及：

```markdown
[[目标笔记]]
[[目标笔记|显示名称]]
```

## Word 文档

墨岛笔记可以把 `.docx` 和旧版 `.doc` 文件导入当前资料库。导入后会在资料库根目录的 `documents/` 中保存一份与外部源文件字节级一致的副本；不会把 Word 转换成 Markdown，也不会改写原文件。

```text
我的资料库/
├── 工作记录.md
├── assets/
│   └── 20260719-a3f82c.png
└── documents/
    ├── 项目方案.docx
    └── 旧版会议记录.doc
```

- 应用会记录导入时的外部源路径，并把之后检测到的源文件变化单向同步到资料库副本。同步方向始终是“外部源文件 → `documents/` 副本”；在 Word、WPS 等软件中请编辑外部源文件，不要把资料库副本当成双向同步文件。
- Word 文件只按文件名和资料库内相对路径参与搜索，当前不索引正文，也不参与 Markdown 双向链接、反向链接、标签或大纲解析。
- `.docx` 可在应用内预览。预览用于快速阅读，字体替换、分页、文本框、公式及其他复杂排版可能与 Microsoft Word 不完全一致；原文件和资料库副本不会因预览而转换或修改。
- `.doc` 不提供应用内预览。选择“使用本地默认软件打开”会交给 macOS 或 Windows 当前为该类型配置的 Word、WPS、LibreOffice 等应用；`.docx` 也始终提供此选项，以便查看完整排版或继续编辑。外部源仍可用时优先打开源文件，让保存结果进入单向同步链路；源文件缺失或文档未关联时改为打开 `documents/` 副本。
- 如果外部源文件被移动、删除或暂时不可访问，现有资料库副本仍保留，但无法继续接收源文件更新。使用“重新关联”选择新的源文件后可继续同步。

Word 能力当前仅面向 macOS 和 Windows 桌面版。未来的 iOS/Android 版本需要重新适配文件选择、沙盒权限、系统“打开方式”和预览组件，不能把桌面行为视为移动端现有能力。

## 数据与安全

对于 Markdown 笔记，`.md` 文件是内容的唯一真实来源。搜索索引、偏好设置和历史快照位于系统应用数据目录，不会把数据库写进资料库。移除资料库只取消登记；删除笔记会进入系统废纸篓。

如果应用检测到文件被其他软件修改，会停止覆盖并让你选择载入外部版本、保留当前内容或另存副本。

建议仍然用 Time Machine、Git 或其他备份工具定期备份资料库。历史快照适合撤销误改，不替代完整备份。

Word 资料库副本同样应纳入资料库备份。应用的单向同步不是云同步，也不替代源文件和资料库的独立备份。
